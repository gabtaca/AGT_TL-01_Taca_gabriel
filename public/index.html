La propriété Display

    Displays an element as an inline element (like span). Any height and width properties will have no effect. This is default.

  • block

    Displays an element as a block element (like p). It starts on a new line, and takes up the whole width

  • contents

  • flex

    Makes the container disappear, making the child elements children of the element the next level up in the DOM

  • grid

    Displays an element as a block-level flex container

  • inline-block

    Displays an element as a block-level grid container

  • inline-flex

    Displays an element as an inline-level block container. The element itself is formatted as an inline element, but you can apply height and width values

  • inline-grid

    Displays an element as an inline-level flex container

  • inline-table

    Displays an element as an inline-level grid container

  • list-item

Les filtres

La propriété CSS filter permet d'appliquer des filtres et d'obtenir des effets graphiques de flou, de saturation, etc. Les filtres sont généralement utilisés pour ajuster le rendu d'une image, d'un arrière-plan ou des bordures.

    Les sortes de filtres applicables

  • blur(5px)
  • brightness(0.4)
  • contrast(200%)
  • drop-shadow(16px 16px 20px blue)
  • grayscale(50%)
  • hue-rotate(90deg)
  • invert(75%)
  • opacity(25%)
  • saturate(30%)
  • sepia(60%)

    • .class{filter:exemple de filtre;}

S

Sélecteurs avancés

  • Dégradés

    Avec linear-gradient et radial-gradient, il est possible de créer des dégradés fluides sans avoir besoin d’images.

  • Ombres portées(voir Box shadows)

    La propriété box-shadow permet d’ajouter des ombres à des éléments, ajoutant de la profondeur et de l’accentuation.

  • Transformations(voir Animtation)

    Les transformations, comme rotate, scale ou translate, permettent de modifier la forme, la taille ou la position d’un élément.

    • Sass est une extension de CSS qui ajoute puissance et élégance au langage de base.

  • Nesting

    Vous pouvez imbriquer des règles CSS à l'intérieur d'autres règles, ce qui permet une meilleure organisation et une meilleure lisibilité du code.

  • Mixins

    Vous pouvez définir des morceaux de code réutilisables qui peuvent contenir des propriétés CSS, des valeurs et même des sélecteurs, ce qui facilite la réutilisation et la gestion du code.

    @mixin et @include

    Ces directives sont utilisées pour créer et inclure des mixins, qui sont des blocs réutilisables de styles. Les mixins peuvent accepter des arguments, ce qui les rend polyvalents pour divers besoins en matière de stylisme.

  • Inheritance

    Sass permet l'héritage des propriétés CSS d'une classe à une autre, ce qui permet de réduire la duplication du code et de rendre les feuilles de style plus modulaires.

    @extend

    Cette directive vous permet de partager un ensemble de propriétés CSS d'un sélecteur à un autre, similaire à l'héritage en programmation orientée objet.

  • Functions

    Vous pouvez définir des fonctions personnalisées pour effectuer des calculs ou des manipulations de chaînes, ce qui permet une plus grande flexibilité dans la génération de styles.

    @for, @each, @while, @function et @return

    Ces directives sont utilisées pour définir et retourner des valeurs à partir de fonctions personnalisées en Sass, permettant des calculs et des transformations plus complexes dans vos feuilles de style.

  • Control Directives

    Sass offre des directives de contrôle telles que @if, @for, et @each qui permettent une plus grande puissance de traitement dans vos feuilles de style.

  • Modular Architecture

    Avec Sass, vous pouvez diviser votre code en fichiers plus petits et les importer au besoin, ce qui favorise une architecture CSS plus modulaire et facile à gérer.

    @import

    Importe un autre fichier Sass ou CSS dans le fichier Sass actuel. Cela vous permet de moduler vos feuilles de style et de les organiser de manière plus structurée.

Z

Z-Index

La propriété z-index définit le « z-order » (NdT : « ordre z » n'est pas usité) d'un élément positionné et de ses éléments enfants ou de ses éléments flexibles. Lorsque des éléments se chevauchent, le z-order détermine l'ordre des différentes couches que formeront les éléments. Généralement, un élément couvrira un autre élément si sa valeur de z-index est supérieure à celle du deuxième élément.

z-index: 1;

@keyframes 50% {{z-index: 5;}}
